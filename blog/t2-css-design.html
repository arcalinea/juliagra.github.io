Relative, absolute, and fixed positioning

July 19, 2015


Positioning elements in CSS

In the early days of the web, designers used html tables for laying out elements. That was a pain, since you had to rewrite the html whenever you wanted to change the layout. Luckily, today we have CSS styles for page layout. Here are some of your options for element positioning: static relative, absolute, and fixed. What's the difference?

Static

This is the default for page elements, and means that it just flows in the page as normal. You'll only use static to override positioning that was set elsewhere, which is fairly rare because positioning is not a property that cascades.

This is where all our boxes start out. Their position right now is determined by their order in the html.

Relative

An element with relative positioning is placed "relative to itself" for the values you provide. For example, if you specify the relative position as "top: 10px;" the element will move 10px down from where it already is. Unlike with absolute positioning, other elements in the flow of the page will still behave like a relatively positioned object has kept its place. Let's take a look at some demo images.

See how the pink box moved 30px away from the left and bottom border of where it used to be, leaving a space? The other boxes politely stay put.

Note: Relative positioning also allows you to use the z-index to set the overlap of an element. A higher number of z-index puts the element on top of other elements.

I gave the first box a z-index of 2, which is higher than 0, the default z-index of all the other boxes I didn't edit. Since I gave it the highest z-index and set its position to relative, it leaps to the top. Without relative positioning, it would not move.

Absolute

An element with absolute positioning can be placed anywhere you want on the page with attributes for top, left, bottom, and right set relative to the browser window. Absolute positioning is good when you want to specify exactly where you want an element on the page, and don't want other elements to interfere with it. It removes the object from the flow of the page, so other elements will act like it's not there.

In the image above, I set the position of the third box to 0px "absolute" from the left and top, so it moves to the upper left corner of the browser window. It wouldn't move from its spot if the position were set to relative. Notice how the other boxes ignore its new position and are covered up.

Fixed

An element with fixed positioning is placed relative to the visitor's browser window, so it will stay put when the page is scrolled. This is usually used for navigation bars or footers. A drawback is that on smaller sized screens, content can be cut off or overlap in awkward ways, so this positioning needs to be thoroughly tested.

I gave box 3a a fixed position that's 50px from the left, 0px from the bottom. Now it's hugging the bottom of the browser window, and will stay put even when we scroll or adjust the screen size.

I scrolled down. Box 3a clings to the bottom of the screen. Perfect for a footer navigation bar.

Inherit

Inherit means your element will take the position of its parent element, no matter what its position values are. (In the tree-like structure of html, parent elements contain child elements that are nested within.)

Box 3a still has instructions to position itself 50px from the left and 0px from the bottom, but setting its position to inherit overrides that and gives it the same position as its parent element, Box 3.

Combining these positioning options in CSS can create a wide variety of page layouts. Other properties, such as float and new values in CSS3, can also change the position of elements, but that's beyond the scope of this post.
