Arrays & Hashes

June 23, 2015


What's the difference?

An array is an ordered data structure, where each entry receives a numbered index that starts from 0. Any object can be stored in an array, but the index is always a number starting from 0 for the 1st item in the array. A hash is an unordered data structure, where instead of a numbered index, each entry receives a "key" that can be a string, a value, or anything you want--even an array can be used as a key. Hashes are sometimes called associative arrays, because their values associate to something other than numbers.

When to use which?

Both arrays and hashes have their uses. An array is good when you have an ordered data set. However, a hash is useful when you have a bunch of data that doesn't come in any particular order, but you need a way to store and reference it.

Say you want to store the latin names of new species of local flora and fauna. You like to add new critters as you come across them, and don't want to play favorites by ordering one before another.
